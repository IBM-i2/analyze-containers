    Updating to the latest version of the analyze-containers repository

    To update your config development environment to use the latest version of the analyze-containers repository, update the images, secrets, and deploy your configs.

    1. Before you update your environment:
      1. Back up any databases that are associated with your configs. For more information, see Back up and restore a development database.
      2. Back up the following directories from your existing copy of the repository:
        • configs
        • dev-environment-secrets
        • connector-images
        • gateway-schemas
        • i2a-data

    2. Download the tar.gz or pull the latest version of the analyze-containers repository from https://github.com/IBM-i2/analyze-containers.

      1. If you download the tar.gz, extract the contents and overwrite your existing copy of the analyze-containers repository. Then, copy your backed up directories from step 1.1 over the new version of the repository.

    3. Update the command line tools in your environment. The latest version of the analyze-containers repository requires XMLStarlet. For more information about installing XMLStarlet, see Command line tools.

    4. To update the images and secrets, run the createDevEnvironment.sh script from the scripts directory.

      ./createDevEnvironment.sh

    5. To update any deployed configs, run the deploy.sh script from the scripts directory with your config name. You must run the script twice, once with the clean task.
      For example:

      ./deploy.sh -c <config_name> -t clean
./deploy.sh -c <config_name>
