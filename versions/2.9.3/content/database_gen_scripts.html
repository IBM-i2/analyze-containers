    Deploying with an external database

    If you are deploying with an external database, you must configure the external database and generate the scripts to create the database.

    1. In your external database system, make a note of the administrator user name and password.
      This user must have permissions to <...>.

    2. Update the analyze-containers config with the details of your external database system:

      1. In the /configs/<config-name>/configuration/InfoStoreNames<db_dialect>.properties file, specify the names of the tablespaces to use in your external system:
        For example:

        # The default tablespace for the Information Store database.
DatabaseTablespace=cstore

# The tablespace for big data tables.
BigDataTablespace=cstore

# The tablespace for indexes on big data tables.
BigIndexTablespace=cstore

# The tablespace for small data tables.
SmallDataTablespace=cstore

# The tablespace for indexes on small data tables.
SmallIndexTablespace=cstore

# Tablespace Locations
cstoreTablespaceLocationDir=/var/lib/postgresql/data/cstore

      2. In the /configs/<config-name>/utils/variables.conf file specify values for the following variables:

        Where:

        • DB_USER_MANAGED specifies whether the config uses an external database. The possible values are true or false.
        • DB_NAME specifies the name of the Information Store database. By default the value is ISTORE.
        • DB_FQDN is the fully-qualified domain name of the external database server.
        • DB_PORT is the port number to connect to the external database.

        • DB_SSL_CONNECTION specifies whether the connection to the external database is secure. The possible values are true or false. When set to true, you must provide the CA's public certificate in the configs/<config-name>/configuration/secrets/<db-dialect>/ directory to ensure trust.

          For example:

          DB_USER_MANAGED="true"
DB_NAME="ISTORE"
DB_FQDN="postgres.eia"
DB_PORT="5432"
DB_SSL_CONNECTION="false"

    3. Provide the administrator's password to analyze containers.
      By default, analyze-containers expects to use the administrator user name described in: Database roles, users, and logins. If your administrator's username is different to this, provide it in the a /configs/<config-name>/utils/variables.conf file.
      The name of the variable depends on the database management system that you're using.
      For example:

      # For PostgreSQL
POSTGRES_USERNAME=

# For SQL Server
SA_USERNAME=
      1. In the configs/<config-name>/configuration/secrets/<db-dialect>/generated/ there is a file for your administrator user. Open the file and paste your administrator password.

    4. To generate the scripts for your config, run the following command:

      deploy -c <config-name> -t generate-db-scripts

      The scripts are generated in the configs/<config-name>/database-scripts/generated directory. Copy the generated scripts to a location where you can run them against your external database system.

    5. On the external database instance, create the database structures and configuration to support the Information Store database:

      1. Create the tablespaces.
        The names of the tablespaces that you create must match the ones that you defined in the InfoStoreNames<db_dialect>.properties file.
      2. Create the pg_cron schedule by using the create_pg_cron_extension.sh script.
      3. Run the runDatabaseCreationScripts.sh script.
      4. Run the runDatabaseScripts.sh script.

      5. Create the roles by using the create_db_roles.sh script.
        The roles in the script align to the roles described in: Database roles, users, and logins.
        The names of the users that are created by analyze-containers must match the names of the roles that are created. If you changed the names of the roles when you created them, update the variables.conf file to provide i2 Analyze with the user names to create. To specify the user names, add variables like the following:

        # For the user in the DBA_Role
DBA_USERNAME=

# For the user in the i2analyze_Role
i2analyze_Role_USERNAME=

    6. In analyze-containers, deploy the config that uses the external database:

      deploy -c <config-name>
